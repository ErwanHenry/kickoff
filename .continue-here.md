### Database Schema Issue

**Current state:** IDs defined with `$defaultFn(() => nanoid())` which may not be compatible with better-auth:

```typescript
// src/db/schema.ts - CURRENT (BROKEN)
export const users = pgTable("users", {
  id: text("id").primaryKey(),  // No default - better-auth should generate
  // ...
});

// Other tables (groups, matches, etc.) also have:
id: text("id").primaryKey(),  // No default - application must generate
```

**Problem:** Drizzle's `$defaultFn` syntax may not be executing properly, causing 500 errors when better-auth tries to create users/accounts.

});
