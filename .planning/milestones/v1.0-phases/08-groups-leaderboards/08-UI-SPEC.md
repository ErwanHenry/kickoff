# Phase 8 — UI Design Contract

> Visual and interaction contract for Groups & Leaderboards phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase Objective:** Organizers can create groups with invite codes, players can view leaderboards ranked by skill, and groups display match history with multiple organizer roles.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | new-york (baseColor: neutral, cssVariables: true, rsc: true) |
| Component library | Radix UI primitives (via shadcn/ui) |
| Icon library | Lucide React (navigation) + Custom FootballIcon (domain concepts) |
| Font | DM Sans (body/headings) + Space Mono (data/labels) |

**Installed Components** (from components.json): avatar, badge, button, card, checkbox, dialog, dropdown-menu, input, label, separator, sonner, tabs, switch, textarea

**Rationale:** Design system established in Phase 1.1. This phase applies existing tokens to group creation, leaderboards, and match history UI.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Touch targets (buttons): Minimum 44x44px for mobile (iOS HIG)
- Leaderboard rows: 48px height for tap-friendly player links
- Invite code copy button: 48px height for easy thumb access

**Source:** Phase 1.1 UI-SPEC, CLAUDE.md mobile-first convention

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 (regular) | 1.5 | DM Sans |
| Label | 14px | 500 (medium) | 1.4 | DM Sans |
| Heading | 24px | 600 (semibold) | 1.2 | DM Sans |
| Display | 32px | 700 (bold) | 1.1 | DM Sans |
| Data/Mono | 14-16px | 400 or 700 | 1.4 | Space Mono |

**Data/Label Mono Usage:**
- Apply `font-mono` to leaderboard rankings (1, 2, 3...), player stats (4.2/5), match counts (12 matchs)
- Apply `font-mono` to invite codes (6-char uppercase display)
- Apply `font-mono` to attendance rates in leaderboard (92%)

**Constraints:**
- Maximum 4 font sizes for scalability
- Maximum 2 font weights (400 regular, 600 semibold) for DM Sans
- Space Mono weights: 400, 700

**Source:** Phase 1.1 UI-SPEC typography contract

## Interaction Patterns

### Group Creation Flow

1. **Navigate to creation**
   - Click "Créer un groupe" from dashboard or groups page
   - Route: `/dashboard/groups/new`

2. **Fill form**
   - Enter group name
   - Slug auto-generates in real-time (debounced 300ms)
   - Validation: Min 3 chars, no special characters

3. **Submit**
   - Click "Créer un groupe"
   - Server generates:
     - UUID for group.id
     - Slug (kebab-case, unique)
     - Invite code (nanoid 6, uppercase)
   - Add creator to group_members with role="captain"
   - Redirect to `/group/[slug]`

4. **Post-creation**
   - Show toast with invite code
   - Highlight copy button animation
   - Suggest: "Partage ce code avec tes potes"

### Invite Code Generation & Sharing

1. **Generation** (GROUP-02)
   - Format: 6-char alphanumeric (nanoid)
   - Uppercase for readability
   - Example: "ABC123"
   - Unique constraint in database

2. **Copy action**
   - Click copy button → Copy to clipboard
   - Show feedback: "Copié !" for 2s
   - Fallback for older browsers: execCommand

3. **Share link** (GROUP-03)
   - URL pattern: `/group/[slug]/join?code={inviteCode}`
   - WhatsApp integration (Phase 10): Pre-filled message
   - QR code generation (optional, Phase 10)

### Group Join Flow (GROUP-04)

1. **Via invite link**
   - User opens link from WhatsApp/email
   - Page pre-fills code from URL
   - Shows group info for context

2. **Code validation**
   - Server checks code exists in database
   - Server checks user is not already member
   - Error: "Code invalide" or "Déjà membre"

3. **Join action**
   - Add user to group_members with role="player"
   - Set joinedAt timestamp
   - Redirect to group page

4. **Success feedback**
   - Toast: "Bienvenue dans [Nom du groupe] !"
   - Show leaderboard (empty if first member)
   - Suggest: "Invite tes potes pour commencer le classement"

### Leaderboard Viewing (GROUP-05, GROUP-06)

1. **Load leaderboard**
   - Query player_stats filtered by groupId
   - Order by avg_overall DESC
   - Limit to group members only

2. **Display rankings**
   - Top 3: Medal emojis + gold/silver/bronze colors
   - 4+: Numbers, neutral color
   - Highlight current user (subtle background)

3. **Player interaction**
   - Tap player row → Navigate to `/player/[id]`
   - View full stats, match history, comments
   - Back button returns to leaderboard

4. **Empty state handling**
   - No ratings yet: Show empty state
   - Suggest: "Jouez un match pour commencer le classement"

### Match History Viewing (GROUP-07)

1. **Load matches**
   - Query matches filtered by groupId
   - Order by date DESC
   - Include played and rated matches only

2. **Display list**
   - Reuse MatchCard component (Phase 2)
   - Add group badge for context
   - Show score if played

3. **Match interaction**
   - Tap match card → Navigate to `/match/[id]`
   - View full details, teams, ratings
   - Back button returns to group

### Role Management (GROUP-08)

1. **View roles**
   - Members list shows role badges
   - Captain/Manager see all roles
   - Players see all roles (read-only)

2. **Change role** (captain/manager only)
   - Open member menu → "Changer le rôle"
   - Select: Capitaine / Manager / Joueur
   - Confirm → Update database
   - Toast: "Rôle mis à jour"

3. **Multiple organizers**
   - Both captain and manager can edit group
   - Both can invite players
   - Both can manage roles
   - Distinction: Captain = creator, Manager = promoted

## Error States

### Invalid Invite Code

**Scenario:** User enters wrong code or malformed code

**Display:**
- Error message above input: "Code d'invitation invalide. Vérifie que tu as le bon code ou demande-le à l'organisateur."
- Input border: Red (destructive color)
- Shake animation on submit
- Button remains enabled (retry)

**Recovery:**
- User can re-enter code
- "Demander un nouveau code" link (optional)

### Already Member

**Scenario:** User tries to join group they're already in

**Display:**
- Message: "Tu es déjà membre de ce groupe."
- Button: "Voir le groupe" (primary, redirects to `/group/[slug]`)

### Group Not Found

**Scenario:** URL slug doesn't exist or group was deleted

**Display:**
- Error page component (404-style)
- Icon: card (outline, 64px, muted)
- Heading: "Groupe non trouvé"
- Body: "Le lien est peut-être invalide ou le groupe a été supprimé."
- CTAs:
  - "Retour à l'accueil" (secondary)
  - "Créer un groupe" (primary)

### Slug Conflict

**Scenario:** User enters name that generates existing slug

**Display:**
- Error below name input: "Ce nom est déjà pris. Essaie un autre nom."
- Slug preview shows red "Indisponible" badge
- "Créer un groupe" button disabled

**Recovery:**
- User modifies name until slug is unique
- Real-time validation (debounced 500ms)
