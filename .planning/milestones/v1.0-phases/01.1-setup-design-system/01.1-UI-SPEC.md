# Phase 01.1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase Objective:** Establish design system foundation — fonts, colors, icons, and tokens for subsequent UI phases.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | new-york (baseColor: neutral, cssVariables: true, rsc: true) |
| Component library | Radix UI primitives (via shadcn/ui) |
| Icon library | Lucide React (navigation) + Custom FootballIcon (domain concepts) |

**Rationale:** Design system already prepared in `addentum/` folder. This phase copies and activates tokens, not creating from scratch.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:** Icon-only touch targets minimum 44x44px (mobile accessibility requirement)

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 (regular) | 1.5 | DM Sans |
| Label | 14px | 500 (medium) | 1.4 | DM Sans |
| Heading | 24px | 600 (semibold) | 1.2 | DM Sans |
| Display | 32px | 700 (bold) | 1.1 | DM Sans |
| Data/Mono | 14-16px | 400 or 700 | 1.4 | Space Mono |

**Data/Label Mono Usage:** Apply `font-mono` to scores (3-2), stats (4.2/5), counters (8/14), timestamps (Mar 15), uppercase labels (CONFIRMÉ, ÉQUIPE A).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAF5` (chalk) | App background — off-white with green tint |
| Secondary (30%) | `#FFFFFF` (chalk-pure) | Cards, elevated surfaces |

| Primary | `#2D5016` (pitch) | Headers, CTAs, primary buttons, navigation |
| Destructive | `#EF4444` (red-card) | No-show status, errors, destructive actions |
| Warning | `#FACC15` (yellow-card) | Waitlist, warnings, attention states |
| Info | `#3B82F6` (whistle-blue) | Links, notifications, info states |
| Team A | `#2D5016` (pitch) | Team A indicators |
| Team B | `#3B82F6` (blue) | Team B indicators |

**Accent Reserved For:**
- RSVP confirmation buttons
- Success toasts
- "Confirmé" status badges
- Positive feedback indicators

**Not for accent:** Navigation, borders, hover states (use pitch light `#4A7A2E` for those)

## Implementation Notes

**Tailwind v4 Configuration:**
- Uses `@import "tailwindcss"` (not PostCSS plugin)
- Theme extensions go in `@theme inline` block in `globals.css`
- No separate `tailwind.config.ts` file (Tailwind v4 convention)

**Font Loading:**
- Use `next/font/google` for optimal loading (display: swap)
- DM Sans: weights 300-700 with optical sizing (opsz 9..40)
- Space Mono: weights 400, 700

**Icon Accessibility:**
- All FootballIcon includes `aria-hidden="true"` (decorative)
- Domain icons get semantic meaning from surrounding context

**Design Token Usage:**
- Import `colors`, `fonts`, `statusBadges`, `attendanceBadge` from `design-tokens.ts`
- Use in components for type-safe color/badge access
- Tailwind classes use same values (defined in `@theme inline`)

**Retrofit Consideration:**
- If existing UI needs updating, use Phase "01.2 — Retrofit Design System"
- Follow replacement rules in GSD_DESIGN_SYSTEM.md section "Tâche GSD : Retrofit Design System"
