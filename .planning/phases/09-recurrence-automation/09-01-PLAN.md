<objective>
Implement Vercel Cron infrastructure and recurring match creation logic. When an organizer creates a match with recurrence="weekly", the system automatically generates the next week's occurrence at midnight daily via Vercel Cron. The new match inherits all parent settings (time, location, player limits, group) but has a new shareToken, status="open", calculated date (+7 days), and NO players auto-confirmed (per RECUR-04). Duplicate occurrences are prevented by querying for existing child matches before creation.

Purpose: Automate weekly match creation for recurring series, eliminating manual organizer work while measuring player reliability through weekly RSVPs.
Output: Working cron endpoint that creates recurring match occurrences, protected by CRON_SECRET, with Vercel configuration.
</objective>

# Key Decisions from CONTEXT.md

**D-01:** Use Vercel Cron Jobs configured in vercel.json with CRON_SECRET env var
**D-02:** Cron schedule runs daily at midnight to check for recurring matches
**D-03:** Use date-fns addWeeks() to calculate next occurrence (+7 days)
**D-04:** New occurrences inherit: location, maxPlayers, minPlayers, groupId, deadline (relative)
**D-05:** New occurrences have different: shareToken (new nanoid), status (always "open"), parentMatchId, date, no players
**D-06:** Create inline Server Action in `src/lib/actions/recurrence.ts` with direct db.insert() queries. Do NOT reuse existing createMatch Server Action because it requires session validation (cron has no session) and creates draft status (recurring matches need status="open" immediately).
**D-07:** Query matches where recurrence="weekly" AND parentMatchId IS NULL (parent matches only)
**D-11:** Protect cron endpoint with CRON_SECRET header validation
**D-12:** Return 401 if secret doesn't match, log failed attempts

# Anti-Patterns from RESEARCH.md

- Don't create cron endpoints without secret validation
- Don't auto-confirm players in recurring matches (violates RECUR-04)
- Don't use relative deadlines incorrectly (must calculate deadline+7 days, not copy)
- Don't send emails for groups with no members (wastes Resend quota) — Plan 02 handles this
- Don't create duplicate occurrences (query for existing child match first)
