Create src/app/api/matches/route.ts with:

1. Add "use server" directive
2. Import: db from @/db, matches schema, auth from @/lib/auth, generateShareToken from @/lib/utils/ids, matchCreateSchema, revalidatePath, redirect
3. Export async function createMatch(input: MatchCreateInput):
   - Get session via auth.api.getSession({ headers: await headers() })
   - Return { error: "Non authentifié" } if no session
   - Validate input with matchCreateSchema.parse(input)
   - Generate shareToken via generateShareToken()
   - Insert match with:
     * All validated fields
     * shareToken (generated)
     * createdBy: session.user.id
     * status: "draft" (per CONTEXT.md D-08)
   - Return created match object
   - RevalidatePath("/dashboard")
   - Redirect to `/match/${match.id}`

4. Export async function publishMatch(matchId: string):
   - Get session and verify user is match creator
   - Update match.status from "draft" to "open"
   - RevalidatePath("/dashboard") and `/match/${matchId}`
   - Redirect to `/m/${shareToken}` (public page)

Per RESEARCH.md "Pattern 1: Server Actions for Mutations" and CONTEXT.md D-08.
  </action>

  </verify>
