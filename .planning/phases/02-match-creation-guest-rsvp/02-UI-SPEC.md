## Component Inventory

### Match Creation Form (src/components/match/match-form.tsx)

**Layout pattern:** Card sections (CONTEXT.md D-01)
- Collapsible/expandable cards for each form section
- Mobile-first vertical stacking
- Progress indicator (step 1/4)

**Fields (from CONTEXT.md D-01 to D-08):**
1. Section "Quand ?"
   - Date picker: `<input type='datetime-local'>` (native, CONTEXT.md D-02)
   - Confirmation deadline: datetime-local with smart default (2h before match, CONTEXT.md D-06)
2. Section "Où ?"
   - Location input: text with autocomplete suggestions (CONTEXT.md D-03)
3. Section "Combien ?"
   - Max players: number input, default 14, range 6-22 (CONTEXT.md D-04)
   - Min players: number input, default 10, range 4-20 (CONTEXT.md D-04)
4. Section "Options"
   - Group association: autocomplete select, optional (CONTEXT.md D-07)
   - Recurrence: select (none/weekly), advanced picker deferred (CONTEXT.md D-05)

**Actions:**
- Primary: "Publier" (creates shareable link, changes status to "open", CONTEXT.md D-08)
- Secondary: "Enregistrer le brouillon" (saves as draft)

### Public Match Page (src/app/m/[shareToken]/page.tsx)

**Layout pattern:** Single column, mobile-first (CONTEXT.md D-09)

**Visual hierarchy & focal points:**
- **Primary visual anchor:** Circular progress ring on player count (accent color #4ADE80, creates urgency with X/Y display)
- **Secondary anchor:** "Je suis là !" RSVP button (sticky at bottom, primary conversion point)
- **Reasoning:** Progress ring immediately shows availability (key decision factor), RSVP button is the main action

**Top information card (CONTEXT.md D-09):**
- Match title (or "Match du [date]" if no title)
- Date + time (formatted: "Mar 15 avril 20h")
- Location (icon + text)
- Player count display (CONTEXT.md D-10):
  - X/Y text (e.g., "8/14 confirmés")
  - Circular progress ring (accent color #4ADE80)
  - Status badge ("Ouvert", "Complet", "Verrouillé")

**Confirmed players list (CONTEXT.md D-11):**
- Scrollable vertical list
- Avatar (initials) + name
- Confirm timestamp ("il y a 2h")
- Max 5 visible initially, "Voir tout" expansion

**Waitlist display (CONTEXT.md D-12):**
- Count only: "3 en liste d'attente"
- No individual names shown publicly
- Appears below confirmed list

### RSVP Button (src/components/match/rsvp-button.tsx)

**States (from CONTEXT.md D-16):**
1. Initial (no guest detected)
   - Input: "Ton prénom ou surnom" (CONTEXT.md D-15)
   - Button: "Je suis là !" (accent color #4ADE80)
   - Touch target: 48px height, full width
2. Confirmed (cookie detected)
   - Display: "Confirmé ✓" (green badge)
   - Button: "Me désinscrire" (destructive color)
   - Personalized banner: "Salut [Prénom] ! Tu es confirmé" (CONTEXT.md D-18)
3. Waitlisted
   - Display: "Liste d'attente" (yellow badge)
   - Text: "Position 3 sur la liste"
   - Button: "Me désinscrire" (destructive color)
4. Full match (CONTEXT.md D-14)
   - Badge: "Complet" (red, destructive color)
   - Waitlist count: "3 en liste d'attente"
   - RSVP button hidden

**Sticky positioning:**
- Fixed at bottom on mobile (safe-area-inset-bottom)
- Below keyboard when input is focused (100dvh max-height)

## Interaction Patterns

### Guest RSVP Flow (from CONTEXT.md D-15 to D-18, RESEARCH.md Pattern 3)

1. **First visit**
   - Guest sees name input + "Je suis là !" button
   - On submit: POST to /api/rsvp with shareToken + guestName
   - Response: guestToken (10-char nanoid)
   - Storage: httpOnly cookie (30 days) + localStorage backup (Safari ITP fallback)

2. **Return visit**
   - Page reads cookie on server render
   - Guest sees personalized banner: "Salut [Prénom] !"
   - RSVP state displayed (confirmed/waitlisted)
   - "Me désinscrire" button available

3. **Cancellation**
   - On "Me désinscrire" click: PATCH to /api/rsvp with action=cancel
   - Database: status → "cancelled"
   - Cookie preserved (guest still recognized for re-RSVP)
   - If waitlist exists: first player auto-promoted (Phase 3)

### Draft + Publish Workflow (from CONTEXT.md D-08)

1. **Draft creation**
   - Form submits with status="draft"
   - No shareToken generated yet
   - Redirects to preview page: `/match/[id]?mode=draft`

2. **Publish action**
   - Organizer clicks "Publier"
   - Server generates shareToken (nanoid 10)
   - Status changes: draft → open
   - Redirects to public page: `/m/[shareToken]`
   - Toast: "Match créé ! Partage ce lien" + copy button

### Waitlist Behavior (from CONTEXT.md D-14, REQUIREMENTS.md GUEST-06, GUEST-07)

1. **Match full**
   - RSVP button hidden
   - "Complet" badge displayed
   - Name input still available (adds to waitlist)

2. **Waitlist join**
   - Guest submits name
   - Status: "waitlisted"
   - Position calculated (current waitlist count + 1)
   - Display: "Position 3 sur la liste d'attente"

3. **Waitlist promotion** (Phase 3, not this phase)
   - Triggered when confirmed player cancels
   - First waitlisted player promoted to confirmed
   - Email notification sent (if user has account)
