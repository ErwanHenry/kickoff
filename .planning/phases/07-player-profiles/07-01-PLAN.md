  <action>Add three new query functions to src/lib/db/queries/players.ts:

1. getPlayerProfile(userId: string, groupId?): Promise<PlayerProfileData>
   - Fetches user by userId
   - Fetches player_stats (prefer groupId-specific, fallback to global where groupId=null)
   - Returns: { id, name, createdAt, matchesPlayed, attendanceRate, avgTechnique, avgPhysique, avgCollectif, avgOverall, totalRatingsReceived, lastMatchDate }
   - Default all fields to 0 for new players (parseDecimal from @/lib/stats)
   - Handle null values gracefully

2. getPlayerMatchHistory(userId: string, limit: number = 10): Promise<MatchHistoryEntry[]>
   - Joins match_players → matches on matchId
   - Filters: userId matches AND status IN ('confirmed', 'no_show')
   - Orders by matches.date DESC
   - Returns array: { matchId, shareToken, date, location, title, status, attended, team, scoreTeamA, scoreTeamB, avgRating }
   - avgRating: Average of ratings received for this match (join ratings where ratedId=userId AND matchId)
   - Limit to 10 records

3. getPlayerComments(userId: string, limit: number = 10): Promise<CommentEntry[]>
   - Queries ratings table where ratedId = userId AND comment IS NOT NULL
   - Joins matches for match context
   - Orders by ratings.createdAt DESC
   - Returns array: { id, comment, createdAt, matchId, matchTitle, matchDate }
   - Limit to 10 records
   - Anonymize: No rater names/IDs exposed

Export types: PlayerProfileData, MatchHistoryEntry, CommentEntry

Use existing patterns: eq, and, desc, limit from drizzle-orm; parseDecimal from @/lib/stats; COALESCE for null handling.
</action>
