  <action>
    Create src/lib/actions/ratings.ts with submitRatings Server Action:

    - Accept formData with matchId, ratings array, raterId (from session or cookie)
    - Auth check:
      - If user logged in: use session.user.id as raterId
      - If guest: read guest_token from cookies, use as raterId
      - If neither: return error "Non authentifié"
    - Validate with ratingSchema.parse()
    - Verify rater participated in match:
      - Query match_players where matchId AND (userId=raterId OR guest_token=raterId) AND attended=true
      - If no record: return error "Tu n'étais pas présent à ce match"
    - Verify match status is "played":
      - If not played: return error "Ce match n'est pas encore terminé"
    - Check for duplicates using UNIQUE constraint (matchId, raterId, ratedId):
      - Query getExistingRatings(matchId, raterId)
      - If rating exists for same ratedId: skip or update (idempotent)
    - Use transaction for atomic insert + stats update:
      - Insert all ratings into ratings table
      - For each unique ratedId: call updatePlayerStats
    - After success, check if match should be marked "rated":
      - Count distinct raters for this match
      - If >= 50% of confirmed players rated: update match status to "rated"
    - Revalidate paths: /match/{matchId}, /match/{matchId}/rate, /m/{shareToken}/rate
    - Return success with count of ratings submitted
  </action>
  <verify>
    <automated>grep -q "submitRatings" src/lib/actions/ratings.ts && grep -q "auth.api.getSession" src/lib/actions/ratings.ts</automated>
  </verify>

    Server Action handles both user (session) and guest (cookie) authentication
    Verifies rater attended the match (attended=true)
    Prevents duplicate ratings via UNIQUE constraint
    Recalculates player stats incrementally
    Updates match status to "rated" when 50% threshold reached
  </done>
