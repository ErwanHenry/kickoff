## Spacing Scale

Declared values (8-point scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Touch targets (buttons): Minimum 44x44px for mobile (iOS HIG)
- Switch toggle: 44x44px minimum for reliable touch input (CONTEXT.md D-16)
- Sticky submit button: 48px height for thumb-friendly tapping (CONTEXT.md D-18)

**Source:** CLAUDE.md mobile-first convention, inherited from Phase 2, CONTEXT.md D-16, D-18

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF (oklch(1 0 0)) | Background, surfaces |
| Secondary (30%) | oklch(0.97 0 0) | Cards, form sections |
| Accent (10%) | #4ADE80 | Sticky submit button, progress indicators |
| Destructive | oklch(0.577 0.245 27.325) | No-show warning, absent player states |
| Primary Brand | #2D5016 | Headers, footer, branded elements |

**Accent reserved for:**
1. "Clôturer le match" sticky submit button
2. Attendance progress indicator ("X/Y joueurs marqués")
3. Present state (green check icon)

**Destructive reserved for:**
1. Absent player state (red x or grayed out)
2. No-show warning text ("Ce joueur recevra le statut 'no_show'")
3. Confirmation dialog warning

**NOT for accent:** Form labels, borders, secondary text, disabled states, score inputs

**Semantic color mapping (from CONTEXT.md):**
- Present → green (accent) → Default state, green check icon
- Absent → gray/desaturated → Grayed out, red x icon, no-show warning

**Source:** globals.css custom kickoff brand colors, CONTEXT.md decisions D-01, D-03

## Component Inventory

### Attendance Form Page (src/app/match/[id]/attendance/page.tsx)

**Layout pattern:** Single column, mobile-first vertical stacking (CONTEXT.md D-17)
- Score section at top
- Attendance list in middle
- Match summary at bottom
- Sticky submit button

**Access:**
- Route: `/match/[id]/attendance`
- Link from: Match detail page ("Clôturer le match" button), dashboard upcoming matches

**Permission:** Organizer-only (match creator only, CONTEXT.md D-12)

### Score Section Card

**Layout pattern:** Highlighted card at top (CONTEXT.md D-04, D-05, D-06)

**Fields:**
1. Heading: "Score du match" (20px semibold)
2. Label: "Équipe A [__] - [__] Équipe B" (14px medium, above inputs)
3. Score inputs:
   - Two native HTML5 number inputs side by side
   - Min value: 0, max value: 99 (no football match exceeds 99)
   - Step: 1
   - Default value: 0-0 (both inputs start at 0)
   - Width: 64px each (sufficient for 2 digits)
   - Center alignment for visual balance

**Validation:**
- Both fields required (CONTEXT.md D-07)
- Block submit until both have values ≥ 0
- Inline error: "Le score est obligatoire"

### Attendance List (src/components/match/attendance-form.tsx)

**Layout pattern:** Toggle list (CONTEXT.md D-01)

**Per-player row:**
- Avatar (initials, 40x40px)
- Player name (16px regular)
- Team badge (A or B, small badge variant)
- Toggle switch (44x44px minimum touch target, CONTEXT.md D-16)

**States (CONTEXT.md D-01, D-02, D-03):**
1. Present (default)
   - Switch: ON (accent color #4ADE80)
   - Visual: Green check icon
   - Name: Normal weight
2. Absent
   - Switch: OFF (gray)
   - Visual: Red x icon or grayed out
   - Name: Strikethrough (optional)
   - Warning: "Ce joueur recevra le statut 'no_show'" (subtle, 12px text-muted-foreground)

**Default state:** All confirmed players marked "Present" by default (CONTEXT.md D-02)

**Quick actions:**
- "Tous présents" button above list
- Single click marks all confirmed players as present
- Useful when everyone showed up

**Progress indicator (CONTEXT.md D-10):**
- Display: "X/Y joueurs marqués"
- Shows: marked count / total confirmed players
- Positioned: Above list, below quick action button
- Updates: Real-time as organizer marks players

### Match Summary Section

**Layout pattern:** Textarea field (CONTEXT.md D-09, D-10)

**Field:**
- Label: "Résumé du match" (optional, 14px medium)
- Placeholder: "Moments forts, MVP, remarques..."
- Component: shadcn/ui textarea
- Max length: 500 characters
- Rows: 3 (mobile), 4 (desktop)
- Resize: Vertical only

**Positioning:** Below score inputs and attendance list, before submit button (CONTEXT.md D-10)

### Submit Button

**Layout pattern:** Sticky at bottom (CONTEXT.md D-18)

**Button:**
- Text: "Clôturer le match"
- Variant: Default (accent color #4ADE80)
- Size: Large (48px height for thumb-friendly tapping)
- Position: Sticky bottom with safe-area-inset-bottom
- Width: Full width on mobile, auto on desktop

**States:**
1. Disabled
   - Condition: Score fields empty OR attendance incomplete
   - Appearance: Muted (gray), no interaction
2. Enabled
   - Condition: Score complete AND all players marked
   - Appearance: Accent color, clickable
3. Loading
   - Appearance: Spinner + "Clôture en cours..."
4. Success
   - Redirect to match detail page
   - Toast: "Match clôturé ! Les joueurs peuvent maintenant noter leurs coéquipiers."

### Confirmation Dialog

**Trigger:** On submit button click (CONTEXT.md D-14)

**Dialog:**
- Title: "Clôturer le match"
- Body: "Tu vas clôturer ce match. Les joueurs absents recevront le statut 'no_show'. Continuer ?"
- Buttons:
  - Cancel: "Annuler" (secondary variant)
  - Confirm: "Oui, clôturer" (destructive variant)

**Pattern:** shadcn/ui dialog component

## Interaction Patterns

### Attendance Marking Flow (from CONTEXT.md D-01 to D-03, D-08)

1. **Page load**
   - All confirmed players displayed in toggle list
   - All players marked "Present" by default (switch ON)
   - Score inputs empty (0-0)
   - Submit button disabled

2. **Marking attendance**
   - Organizer taps toggle switch to change player status
   - Present → Absent: Switch OFF, show no-show warning
   - Absent → Present: Switch ON, hide warning
   - Progress indicator updates: "X/Y joueurs marqués"

3. **Entering score**
   - Organizer taps score input fields
   - Native number keyboard appears on mobile
   - Both fields required for validation

4. **Optional summary**
   - Organizer taps textarea field
   - Types match summary (max 500 chars)
   - Character counter appears: "X/500"

5. **Submission**
   - Organizer taps "Clôturer le match" button
   - Confirmation dialog appears
   - Confirm → Loading state → Redirect to match detail
   - Toast: "Match clôturé ! Les joueurs peuvent maintenant noter leurs coéquipiers."

### Validation Flow (from CONTEXT.md D-07, D-08)

1. **Score validation**
   - Both score fields required
   - Values must be ≥ 0
   - Error: "Le score est obligatoire" (inline below inputs)

2. **Attendance validation**
   - All confirmed players must be marked
   - Error: "Tous les joueurs doivent être marqués présents ou absents"
   - Submit button blocked until validation passes

### Post-Close Flow (from CONTEXT.md D-15)

1. **Database update**
   - Match status: current → "played"
   - Score saved: score_team_a, score_team_b
   - Match summary saved: match_summary (if provided)
   - Absent players: attended = false, status = "no_show"
   - Present players: attended = true

2. **Redirect**
   - Destination: Match detail page `/match/[id]`
   - Toast notification: "Match clôturé ! Les joueurs peuvent maintenant noter leurs coéquipiers."

3. **Next phase setup**
   - Match now eligible for player ratings (Phase 6)
   - Players can rate teammates who participated
   - No-show players excluded from rating flow

## Mobile-First Constraints

From CLAUDE.md, CONTEXT.md D-16, D-17, D-18:

| Viewport | Target |
|----------|--------|
| iPhone SE (375px) | Minimum supported width |
| iPhone 14 (390px) | Primary target |
| Android mid-range (412px) | Secondary target |

**Touch requirements:**
- Minimum touch target: 44x44px (iOS HIG)
- Switch toggle: 44x44px minimum (CONTEXT.md D-16)
- Submit button: 48px height (thumb-friendly, CONTEXT.md D-18)
- Spacing between interactive elements: 8px minimum

**Viewport handling:**
- Dynamic viewport units (100dvh) for mobile keyboard
- Safe-area-inset for iOS notched devices
- Sticky submit button accounts for virtual keyboard (CSS env() variables)
- Single column layout on mobile (CONTEXT.md D-17)

**Mobile-specific layout:**
- Score inputs: Stack vertically on very small screens (375px)
- Attendance list: Full width, no horizontal scroll
- Sticky button: Below keyboard when textarea focused (100dvh max-height)

## Accessibility

From shadcn/ui Radix primitives, WCAG 2.1 AA:

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Radix UI handles automatically (switch, textarea, number inputs) |
| Screen reader support | ARIA labels on all interactive elements |
| Focus indicators | Visible focus ring (outline-ring/50 in globals.css) |
| Color contrast | Minimum 4.5:1 for text, 3:1 for large text |
| Touch targets | Minimum 44x44px (iOS HIG) |

**Specific patterns:**
- Submit button: aria-label="Clôturer le match et enregistrer la présence"
- Toggle switch: aria-label="Marquer {player_name} comme présent ou absent"
- Progress indicator: aria-live="polite" for attendance progress
- Form inputs: Associated labels (no placeholder-only labels)
- Error messages: Inline with aria-describedby
- Confirmation dialog: aria-describedby for warning message

**Switch component accessibility:**
- aria-checked: true/false for present/absent state
- Role: switch
- Keyboard: Space/Enter to toggle
